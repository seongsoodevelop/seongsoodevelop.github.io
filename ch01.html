Chapter 1: 컴퓨터 네트워크와 인터넷

1.1 인터넷이란?

인터넷은 전 세계 수십억 개의 장치들이 연결된 네트워크들의 네트워크(network of networks)이다. 각 장치는 호스트(host) 또는 종단 시스템(end system)이라 불리며, 웹 브라우저, 이메일 클라이언트 같은 다양한 응용 프로그램(application)을 실행한다.

인터넷의 핵심 구성 요소

  • 통신 링크(communication links): 광섬유, 구리선, 무선, 위성 등을 통해 데이터 전달
  • 라우터(router) 및 스위치(switch): 데이터 경로 결정과 패킷 전달을 담당
  • 대역폭(bandwidth): 통신 링크의 전송 속도, 초당 비트 수로 표현됨

인터넷 구조

인터넷은 다양한 수준의 ISP(Internet Service Provider)가 계층적으로 연결되어 구성된다:

  • 로컬 ISP ↔ 지역 ISP ↔ 글로벌 ISP
  • IXP(Internet Exchange Point)를 통해 ISP 간 피어링(peer링) 수행
  • 콘텐츠 제공자(예: Google)는 자체 데이터 센터와 네트워크를 통해 직접 콘텐츠 전달

인터넷의 관점

  • "nuts and bolts" 관점: 물리적 하드웨어와 프로토콜 중심의 기술적 시각
  • 서비스 관점: 사용자가 다양한 응용 서비스를 활용할 수 있는 플랫폼으로서의 역할

서비스 관점에서는 개발자가 인터넷 기능을 활용할 수 있도록 API(Application Programming Interface)를 통해 기능을 제공한다.

네트워크의 분류

  • LAN (Local Area Network): 근거리 통신망 (예: 가정, 학교)
  • MAN (Metropolitan Area Network): 도시 규모의 통신망
  • WAN (Wide Area Network): 광역 통신망, 대표적으로 인터넷

네트워크 예시: 가정용 WiFi, 회사 LAN, 스마트폰이나 노트북을 통한 이동 네트워크

IP 주소와 DNS

  • IP 주소: 장치가 인터넷에 접속하기 위한 고유 주소
  • DNS (Domain Name System): 도메인 이름을 IP 주소로 변환하여 네트워크 상에서 장치들을 식별 가능하게 함

인터넷과 IoT (사물인터넷)

최근에는 센서, 스마트폰, 가전기기 등이 인터넷에 연결되는 IoT (Internet of Things)가 확산되고 있으며, 이는 인터넷의 경계를 기존 컴퓨터에서 실세계 사물로 확장시키고 있다.

예시: 스마트 냉장고, 웨어러블 기기, 스마트 전등 등

인터넷 어플리케이션

WWW (World Wide Web)은 가장 대표적인 인터넷 응용 서비스이며, 그 외에도 다양한 앱이 인터넷을 기반으로 동작한다.

  • 메신저 (예: 카카오톡)
  • 전자상거래 (예: 쿠팡)
  • 모바일 결제 (예: 삼성페이)
  • 가상화폐, 가상 공간, 클라우드 서비스 등

1.2 네트워크의 가장자리 (Network Edge)

네트워크 가장자리는 종단 시스템(end system), 즉 호스트(host)가 위치하는 부분이다. 이는 클라이언트와 서버를 포함하며, 주로 사용자의 컴퓨터, 스마트폰, 웹 서버 등이 해당한다.

종단 시스템(End Systems)

  • IP 주소를 통해 식별됨
  • 응용 프로그램에 따라 클라이언트와 서버 역할을 모두 수행할 수 있음
  • 클라이언트: 요청을 보내는 주체 (예: 웹 브라우저, 메일 클라이언트)
  • 서버: 요청을 처리하고 응답을 제공하는 주체 (예: 웹 서버, 이메일 서버)
  • 클라이언트/서버는 하드웨어이자 소프트웨어 개념 모두 포함

노드(Node)

네트워크에 연결된 모든 기기를 의미하며, 프린터 등 비PC 장치도 포함됨. 즉, 호스트 ⊆ 노드이다.

네트워크 중심 장비

  • 라우터(Router): 네트워크 간 데이터 경로를 결정하고 패킷을 전달하는 장치
  • 라우터는 인터넷의 중심에 위치하며, 네트워크 간 연결(즉, 인터넷)을 가능하게 함

접속 네트워크 (Access Network)

종단 시스템이 인터넷에 접속하기 위해 사용하는 네트워크. 접속 방식에 따라 다음과 같이 나뉨:

  • 가정용 네트워크
    • DSL (Digital Subscriber Line): 전화선을 사용하여 중앙국의 DSLAM에 연결
    • 케이블 모뎀: 동축 케이블을 사용, 주파수 분할 방식. CMTS에 연결됨
    • FTTx (Fibre to the X): 광케이블이 어디까지 도달하는지에 따라 구분됨
      • FTTN: 전신주나 거리까지
      • FTTC: 특정 지역 노드까지
      • FTTB: 건물까지
      • FTTH: 가정까지 직접 연결 (가장 빠르고 안정적)
  • 기업 네트워크
    • 이더넷 기반 고속 연결 (1~10 Gbps)
    • 스위치(switch)를 통해 여러 장치가 네트워크에 연결됨
    • Access Point를 통해 무선 접속도 가능
  • 무선 네트워크
    • WiFi (802.11): 건물 내부에서 사용
    • 모바일 네트워크 (3G, 4G, 5G): 셀룰러 기반, 수 km 범위 커버
    • 위성 통신: 광역 커버, 높은 지연 시간

전송률 및 공유 여부

  • 전송률 (Transmission Rate): 링크가 초당 전송할 수 있는 최대 비트 수 (예: 100 Mbps, 1 Gbps)
  • 전용 vs. 공유 링크: DSL은 전용 회선, 케이블망은 이웃 사용자와 공유

물리 매체 (Physical Media)

  • 유도 매체 (Guided Media)
    • 꼬임쌍선 (Twisted Pair): Cat5 (100 Mbps~1 Gbps), Cat6 (10 Gbps)
    • 동축 케이블 (Coaxial): 광대역, 양방향, HFC 방식에서 사용
    • 광섬유 (Fiber Optic): 빛의 펄스로 데이터 전송, 고속·낮은 오류율
  • 비유도 매체 (Unguided Media)
    • 무선 신호는 공기 중 전파를 통해 전달됨
    • 유형: 무선 LAN (WiFi), 셀룰러, 지상 마이크로파, 위성 통신 등

데이터 패킷 전송

네트워크에서는 데이터를 패킷(packet)이라는 단위로 전송한다. 이는 데이터를 작은 단위로 나눈 것이며, 전송 매체를 통해 노드 간에 전달된다.

전송 속도는 전송률(bandwidth)에 의해 결정되며, 패킷은 라우터를 거쳐 목적지로 전달된다.

보충

기본 용어 정리

  • 디지털 신호 vs 아날로그 신호
    - 디지털 신호: 0과 1의 이산적인(discrete) 값으로 표현됨. 컴퓨터가 처리하는 모든 정보는 디지털 형태로 변환됨.
    - 아날로그 신호: 연속적인 파형으로 구성되어 있으며, 시간의 흐름에 따라 값이 연속적으로 변함 (예: 소리, 빛, 온도 등 자연 현상)
    디지털 신호는 노이즈에 강하고 복원 용이, 아날로그는 자연스러운 신호 표현에 유리함.
  • 단위 변환
    - 데이터 저장 및 전송 단위는 다음과 같은 단계로 증가함:
    Bit → Byte (8 bits) → Kilobyte (KB, 103) → Megabyte (MB) → Gigabyte (GB) → Terabyte (TB) → Petabyte (PB) → Exabyte (EB) → Zettabyte (ZB) → Yottabyte (YB)
    - 1 Byte = 8 bits, 즉 'a' 같은 문자 하나도 컴퓨터 내에서는 8bit로 표현됨.
  • Hz (헤르츠)
    - 주파수의 단위로, 1초 동안 일어나는 진동의 횟수를 나타냄.
    - 예: 60Hz 신호는 1초에 60번 진동함 → 전기, 무선 통신에서 자주 사용됨.
  • bps (bits per second)
    - 초당 몇 비트의 데이터를 전송할 수 있는지를 나타내는 전송 속도 단위.
    - 예: 100 Mbps는 1초에 1억 비트를 전송 가능.
  • Bandwidth (대역폭)
    - 네트워크 링크 또는 시스템이 얼마나 많은 데이터를 동시에 전송할 수 있는지를 나타냄.
    - 단위는 보통 bps이며, 전송 속도와 비례함.
    - **속도와 동일한 개념은 아님**: 대역폭은 용량(capacity), 속도는 실제 처리된 양.
    예: 고속도로 차선 수 = 대역폭, 실제 차량 흐름 속도 = 전송률

1.3 네트워크 중심 (Network Core)

네트워크 중심(Network Core)은 인터넷의 중추 역할을 하며, 수많은 라우터(router)와 통신 링크들로 구성된다. 이 구조는 패킷이 출발지에서 목적지로 전송되는 동안 경로를 설정하고 데이터를 중계하는 핵심 기능을 수행한다.

1. 패킷 교환 방식 (Packet Switching)

  • 데이터는 패킷(packet)이라는 작은 단위로 분할되어 전송된다.
  • 각 패킷은 독립적으로 경로를 따라 목적지로 이동하며, 중간 경로에서 다른 패킷들과 혼합될 수 있다.
  • 저장 후 전달(store-and-forward) 방식으로, 각 라우터는 전체 패킷을 받은 뒤에 다음 노드로 전달한다.
  • 전송 지연은 다음과 같은 요소의 합으로 계산된다:
    • 전송 지연: 패킷을 링크에 올리는 데 걸리는 시간
    • 전파 지연: 전기 신호가 링크를 따라 이동하는 데 걸리는 시간
    • 처리 지연: 패킷을 검사하고 결정하는 시간
    • 큐잉 지연: 패킷이 라우터의 큐에서 대기하는 시간
  • 네트워크가 혼잡하면 큐잉 지연이 증가하고, 큐가 가득 차면 패킷 손실이 발생할 수 있다.

2. 회선 교환 방식 (Circuit Switching)

  • 통신 시작 전에 종단 간 경로를 예약하는 방식
  • 전화망처럼 오랜 시간 일정한 데이터 흐름이 필요한 경우에 적합
  • 전송 중에는 대역폭이 독점적으로 할당되어 안정적이나, 사용하지 않을 때에도 자원이 낭비됨
  • FDM (Frequency Division Multiplexing): 주파수를 분할하여 여러 사용자에게 동시에 제공
  • TDM (Time Division Multiplexing): 시간을 분할하여 사용자에게 번갈아 제공

3. 비교: 패킷 교환 vs. 회선 교환

기준 패킷 교환 회선 교환
자원 할당 공유 전용
설정 시간 필요 없음 연결 설정 필요
성능 혼잡 시 지연 가능성 고정 대역폭으로 안정적
효율성 높음 (자원을 필요할 때만 사용) 낮음 (자원 낭비 발생 가능)
적합한 경우 불규칙적 트래픽 (인터넷) 지속적 트래픽 (음성 통화)

4. 네트워크 구조 (Internet Structure)

인터넷은 전 세계 수많은 ISP(Internet Service Provider)와 그들이 연결된 네트워크의 계층 구조로 이루어져 있다. 각 ISP는 고객 장치를 인터넷에 연결하는 역할을 하며, 이들은 서로 연결되어 네트워크의 네트워크를 형성한다.

  • Tier-1 ISP: 전 세계적인 백본 네트워크 제공, 모든 네트워크에 접근 가능 (예: AT&T, NTT)
  • IXP (Internet Exchange Point): ISP 간에 트래픽을 교환하기 위한 물리적 접속 지점
  • Peering Link: 두 네트워크 간의 직접적인 상호 연결
  • CPN (Content Provider Network): Google, Microsoft 등은 자체 글로벌 네트워크(CDN 포함)를 구축하여 사용자에게 직접 콘텐츠를 전송

인터넷 연결 방법의 발전 과정

  1. 모든 ISP가 서로 직접 연결 → 비현실적 (복잡도 O(N²))
  2. 중앙 집중식 글로벌 ISP를 통해 연결 → 독점 문제 발생
  3. 여러 글로벌 ISP를 통해 Peering 및 IXP 구성 → 확장성과 분산 구조 확보
  4. 콘텐츠 제공자들이 자체 네트워크 운영 → 속도 개선, 비용 절감

이러한 구조 덕분에 인터넷은 확장성, 유연성, 비용 효율성을 확보하며, 전 세계 수십억 대의 장치가 서로 신속하고 안정적으로 데이터를 교환할 수 있다.

1.4 네트워크에서의 지연, 손실, 처리율

패킷이 송신지에서 수신지로 전달되는 동안 다양한 지연(delay) 요인이 발생하며, 네트워크 혼잡 시에는 데이터 손실(packet loss)도 일어날 수 있다. 또한 송수신 간의 실제 전송률인 처리율(throughput)은 네트워크 성능을 평가하는 핵심 지표이다.

1. 지연(delay)의 네 가지 요소

  • 1) 처리 지연 (Nodal Processing Delay)
    - 라우터가 패킷의 헤더를 검사하고 출력 링크를 결정하는 데 걸리는 시간
    - 에러 검사 등도 포함됨 (μs 수준)
  • 2) 큐잉 지연 (Queuing Delay)
    - 패킷이 출력 링크로 나가기 위해 라우터의 큐(buffer)에 대기하는 시간
    - 혼잡도에 따라 달라지며, 예측 불가능
    - 트래픽 강도: La / R (L: 패킷 길이, a: 평균 패킷 도착률, R: 링크 대역폭)
    • La/R → 0 에 가까울수록 지연 작음
    • La/R → 1 에 가까울수록 지연 커짐
    • La/R > 1 → 패킷 손실 발생 (처리 불가 상태)
  • 3) 전송 지연 (Transmission Delay)
    - 패킷을 전송 링크에 밀어 넣는 데 걸리는 시간
    - 계산 공식: L / R (L: 비트 수, R: bps)
    - 예: 1,000비트 패킷을 1 Mbps 링크에 전송 시 1ms 소요
  • 4) 전파 지연 (Propagation Delay)
    - 신호가 매체를 따라 전파되는 시간
    - 계산 공식: d / s (d: 거리, s: 전파 속도)
    - 전파 속도 예: 구리선 약 2×108 m/s, 광섬유는 더 빠름

2. 패킷 손실 (Packet Loss)

출력 큐가 가득 찬 경우, 도착하는 패킷은 폐기되며 손실이 발생한다. 이때의 처리 방식은 전송 프로토콜에 따라 달라진다.

  • TCP: 손실 감지 시 재전송 수행 (신뢰성 보장)
  • UDP: 재전송 없이 손실 허용 (실시간 스트리밍 등에서 사용)

3. 처리율 (Throughput)

  • 순간 처리율: 특정 순간에 송신측에서 수신측으로 전달되는 비트 수
  • 평균 처리율: 일정 시간 동안의 전체 평균 전송률
  • 병목 링크(Bottleneck Link): 경로 상에서 가장 처리율이 낮은 링크가 전체 처리율을 결정함

병목 조건 예시:

  • 송신률 RS > 병목 링크 용량 RC → 처리율은 RC
  • 송신률 RS < 병목 링크 용량 RC → 처리율은 RS
  • 실제 네트워크에서는 중간 링크마다 제한이 존재하므로 복합 병목 상황 발생

4. 트래픽 강도와 큐잉 지연 관계

평균 트래픽 강도 La/R이 0에 가까우면 지연은 작지만, 1에 가까워질수록 지연이 급격히 증가한다.
La/R > 1이면 큐는 감당할 수 없고 손실이 발생하므로 설계 시 중요 고려 요소이다.

5. traceroute 도구

traceroute는 각 중간 라우터까지의 왕복 시간(RTT)을 측정하여, 실제 인터넷 경로 상의 지연과 손실 현상을 분석하는 데 사용된다.
여러 패킷을 전송하면서 TTL(Time To Live)을 조절해 각 홉(hop)의 지연을 추적함.

1.5 프로토콜 계층과 서비스 모델

인터넷은 다양한 기능과 시스템이 유기적으로 작동하는 복잡한 구조이기 때문에, 이를 체계적으로 관리하기 위해 계층적 모델(layered architecture)이 사용된다. 각 계층은 독립적으로 설계되어 모듈화가 가능하고, 복잡성을 줄이며 유지보수를 용이하게 한다.

1. 계층화의 목적

  • 복잡한 기능을 논리적으로 분리하여 이해와 구현을 단순화한다.
  • 모듈화를 통해 설계 변경이 다른 계층에 영향을 주지 않도록 한다.
  • 인터페이스와 책임이 명확해져서 시스템의 확장성 및 유연성이 향상된다.

비유적 예시:

  • 항공 시스템: 예약 시스템(응용)과 탑승 절차(운송)는 독립적이므로, 한 부분을 변경해도 전체 시스템에 영향을 주지 않음
  • 운영체제: 하드웨어 → 커널 → 응용 프로그램처럼 계층화 구조

2. 인터넷 프로토콜 스택 (Internet Protocol Stack)

인터넷은 총 5개의 계층으로 구성되어 있으며, 각 계층은 다음과 같은 역할을 수행한다:

  • 1) 응용 계층 (Application Layer)
    - 네트워크 기반 응용 프로그램들 간의 정보 교환을 담당
    - 예: HTTP(웹), FTP(파일), SMTP(메일), DNS(도메인)
  • 2) 전송 계층 (Transport Layer)
    - 종단 시스템 간의 신뢰성 있는 데이터 전송
    - 흐름 제어, 오류 제어, 혼잡 제어 수행
    - 예: TCP(신뢰성 보장), UDP(빠른 전송)
  • 3) 네트워크 계층 (Network Layer)
    - 패킷을 목적지까지 전달할 경로를 설정 (라우팅)
    - 예: IP(주소 지정), ICMP(오류 및 제어)
  • 4) 링크 계층 (Link Layer)
    - 인접한 장치 간의 데이터 프레임 전송
    - 에러 검출 및 재전송 관리 포함
    - 예: Ethernet, PPP, Wi-Fi
  • 5) 물리 계층 (Physical Layer)
    - 0과 1의 비트를 전기적, 광학적 신호로 변환하여 전송
    - 예: 케이블, 라디오, 광섬유 등

3. OSI 7계층 모델 (ISO/OSI Reference Model)

OSI(Open Systems Interconnection)는 7개의 계층으로 정의된 모델로, 인터넷 프로토콜 스택의 개념적 기반이 되었다. 아래는 인터넷 프로토콜 스택에 추가되는 2개 계층이다.

  • 표현 계층 (Presentation Layer): 데이터 포맷 변환, 인코딩/디코딩, 암호화 등
    예: ASCII, MPEG, JPEG
  • 세션 계층 (Session Layer): 세션 생성, 유지, 복구 담당
    예: TLS, SSH, RPC
  • 인터넷에서는 이 두 계층이 독립적으로 구현되지 않으며, 필요 시 응용 계층에서 처리

4. 캡슐화 (Encapsulation)

상위 계층의 데이터는 하위 계층으로 내려가면서 헤더 정보가 추가되며 전송된다. 이 과정을 캡슐화라 한다. 수신 측에서는 반대로 헤더를 하나씩 제거 (역캡슐화)하며 데이터를 처리한다.

  • 응용 계층 데이터 → 전송 계층이 TCP/UDP 헤더 추가
  • → 네트워크 계층이 IP 헤더 추가
  • → 링크 계층이 프레임 헤더 추가
  • → 물리 계층은 비트 단위로 변환하여 전송

5. 계층화의 장단점

  • 장점: 설계 및 디버깅 용이, 독립적 개발 가능, 구조적 이해 쉬움
  • 단점: 캡슐화로 인한 중복 정보 및 오버헤드, 계층 간 경계의 비효율

계층화된 프로토콜 모델은 시스템 설계자와 개발자에게 명확한 인터페이스와 책임을 제공하며, 오늘날의 인터넷이 확장성과 안정성을 유지할 수 있도록 하는 핵심 구조이다.

1.6 네트워크 보안

네트워크 보안은 외부 공격으로부터 데이터와 시스템을 보호하기 위한 기술과 절차의 집합이다. 인터넷은 처음 설계될 당시 신뢰 기반의 폐쇄형 구조를 가정하였기 때문에, 현재의 개방된 환경에서는 다양한 보안 위협에 취약하다.

1. 보안의 기본 질문

  • 악의적인 사용자는 컴퓨터 네트워크를 어떻게 공격하는가?
  • 사용자는 이러한 공격을 어떻게 방어할 수 있는가?
  • 공격을 예방할 수 있는 구조는 어떻게 설계되어야 하는가?

2. 악성코드 (Malware)

말웨어는 컴퓨터 시스템에 악의적으로 침투하는 소프트웨어로, 다음과 같은 유형이 존재한다:

  • 바이러스 (Virus): 실행 파일에 첨부되어 복제되는 형태이다.
  • 웜 (Worm): 스스로 네트워크를 통해 확산되는 독립형 프로그램이다.
  • 스파이웨어 (Spyware): 사용자의 입력, 방문 웹사이트, 데이터 등을 무단 수집하여 외부로 전송한다.
  • 봇넷 (Botnet): 말웨어에 감염된 다수의 호스트로 구성된 네트워크로, 스팸 메일 전송이나 DDoS 공격 등에 사용된다.

3. 서비스 거부 공격 (Denial of Service, DoS)

서비스 거부 공격은 대상 서버나 네트워크 자원에 과도한 요청을 보내 정상적인 서비스를 방해하는 공격이다. 일반적으로 다음과 같은 절차로 진행된다:

  1. 공격 대상 식별
  2. 봇넷을 통해 다수의 좀비 호스트 확보
  3. 대상에 집중적으로 트래픽 전송

DDoS는 분산 서비스 거부 공격으로, 더욱 위협적이다.

4. 스니핑 (Sniffing)

공유 매체(Ethernet, 무선 네트워크)에서는 패킷이 모든 호스트에 도달하기 때문에, 네트워크 인터페이스 카드(NIC)가 ‘무차별 모드’로 설정된 경우, 지나가는 모든 패킷을 가로채 기록할 수 있다. 이를 스니핑이라 한다.

5. IP 스푸핑 (IP Spoofing)

공격자는 출발지 IP 주소를 위조하여 자신을 숨기거나, 응답을 다른 사용자에게 유도할 수 있다. 이는 회신 주소를 신뢰하는 시스템이나 프로토콜에서 큰 위협이 된다.

6. 보안 설계의 필요성

현대 인터넷은 다양한 위협에 노출되어 있기 때문에, 보안은 모든 계층에서 고려되어야 한다. 프로토콜 설계 시 암호화, 인증, 무결성 검증 등을 포함하는 것이 필요하다.

본 장에서는 보안의 개요만 다루며, 심화된 내용은 8장에서 논의된다.

1.7 인터넷의 역사

인터넷은 1960년대부터 시작되어 수십 년에 걸쳐 발전해 온 분산형 통신 시스템이다. 초기에는 군사적 목적의 연구에서 출발하였으며, 이후 학술, 상업, 대중 사용으로 확산되었다.

1. 초기 개념과 ARPANET (1960~1970년대)

  • 1961년: Kleinrock이 큐잉 이론을 통해 패킷 교환의 효율성을 입증함
  • 1969년: 미국 국방부 고등연구계획국(ARPA)이 ARPANET을 구축함
  • 1972년: ARPANET의 공개 시연 및 최초의 이메일 프로그램 등장

2. TCP/IP와 인터넷의 기틀 (1980년대)

  • 1983년: TCP/IP가 공식적으로 채택되어 ARPANET에 적용됨
  • 이후 DNS, SMTP, FTP 등 주요 프로토콜이 정의됨
  • BITNET, CSNET, NSFNET 등의 국가 단위 네트워크가 확대됨

3. 웹과 상업화의 시작 (1990년대)

  • 1991년: 월드 와이드 웹(WWW) 개념 발표 (Tim Berners-Lee)
  • 1994년: Mosaic/Netscape 브라우저 출시로 대중화 가속
  • ARPANET 퇴역, NSFNET의 상업화 → 오늘날 인터넷 기반 마련

4. 대규모 확장과 모바일 인터넷 (2000년대 이후)

  • 모바일 기기의 보급과 함께 무선 인터넷 사용 급증
  • Google, Facebook, Amazon 등의 플랫폼 기업 부상
  • 클라우드 컴퓨팅, CDN, IoT, 소셜 네트워크 서비스 확산
  • 2020년대: 5G, 엣지 컴퓨팅, AI 기반 네트워크로 진화 중

인터넷은 단순한 통신망을 넘어, 인간 활동 전반을 지탱하는 핵심 인프라로 자리 잡았으며, 지금도 끊임없이 진화하고 있다.
