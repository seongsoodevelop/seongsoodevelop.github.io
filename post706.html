Mobile IP 개요와 필요성

Mobile IP는 IP 계층에서 노드의 이동성을 지원하여, 변하지 않는 홈 주소로 지속적인 통신을 가능하게 합니다. 홈 에이전트를 통한 간접 라우팅 구조를 기반으로 하며, 기존 세션을 유지하면서도 네트워크를 이동할 수 있도록 설계되었습니다.

Mobile IP의 구성 요소

Mobile IP는 모바일 노드(MN), 홈 에이전트(HA), 외부 에이전트(FA)로 이루어집니다. MN은 이동 장치이고, HA는 홈 네트워크에서 노드 위치를 관리하며, FA는 방문 네트워크에서 COA를 제공하고 HA와 연결을 중계합니다.

이동 노드의 위치 등록 과정

모바일 노드는 방문 네트워크에서 FA 광고를 수신하고, 등록 요청을 보내어 COA를 HA에 등록합니다. HA는 이를 기록하고 등록 응답을 반환하여 모바일 노드의 위치 관리가 완료됩니다.

패킷 전달 과정 (터널링과 캡슐화)

상대 노드가 홈 주소로 패킷을 보내면 HA는 이를 수신하여 COA로 캡슐화해 터널링하고, FA가 디캡슐화하여 모바일 노드에 전달합니다. 응답은 MN에서 직접 상대에게 전송됩니다.

삼각 라우팅 문제와 최적화

삼각 라우팅은 HA를 경유하면서 경로가 비효율적인 문제를 뜻합니다. 최적화 방안으로는 HA가 상대에게 COA를 알려 직접 경로로 전송하게 하는 바인딩 업데이트 기법이 있습니다.

요약 및 학습 팁

요약: Mobile IP는 홈 에이전트를 통해 위치 투명성을 제공하며, 터널링으로 패킷을 전달하고 최적화를 통해 직접 경로 전송도 지원합니다.

학습 팁: Mobile IP 흐름을 순서도로 그려보고, 메시지 종류와 내용을 표로 정리하며, IPv6와 비교하거나 실제 적용 사례를 찾아보면 학습에 도움이 됩니다.
