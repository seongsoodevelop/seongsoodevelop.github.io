모바일 사용자의 주소 설정 및 라우팅 원리

목차

  • 이동 환경에서의 주소 지정 문제
  • 홈 네트워크와 방문 네트워크 개념
  • 간접 라우팅 방식 (홈 에이전트를 통한 통신)
  • 직접 라우팅 방식 (송신자의 위치 파악 및 통신)

이동 환경에서의 주소 지정 문제

이동 중 IP 주소가 네트워크 변경에 따라 바뀌면 기존 세션이 끊기는 문제가 발생합니다. 이를 해결하기 위해 주소를 고정하고 네트워크가 전달을 책임지거나, 주소가 바뀔 때마다 송신자에게 새 주소를 알리는 방식이 사용됩니다. 각각은 간접, 직접 라우팅 방식과 연결됩니다.

홈 네트워크와 방문 네트워크 개념

홈 네트워크는 모바일 노드의 영구 IP를 부여한 네트워크이며, 방문 네트워크는 현재 접속 중인 다른 네트워크입니다. 홈 에이전트는 노드의 이동 상태를 관리하고, 케어-오브-주소(COA)는 방문지 임시 주소로 사용됩니다.

간접 라우팅 방식

송신자가 이동 노드의 홈 주소로 패킷을 보내면 홈 에이전트가 이를 수신하여 현재 COA로 캡슐화하여 전달합니다. 송신자 입장에선 노드가 이동했는지 알 필요가 없으며, 홈을 거치는 삼각 라우팅 경로가 형성됩니다.

직접 라우팅 방식

송신자가 홈 에이전트로부터 이동 노드의 COA를 받아 이후 직접 해당 주소로 패킷을 보냅니다. 경로가 최적화되지만 송신자가 노드 위치를 인지하고 따라야 하며, 동기화가 복잡해집니다.

요약 및 학습 팁

요약: 모바일 사용자의 주소와 라우팅은 홈/방문 네트워크 구분과 간접/직접 방식의 차이로 이해할 수 있습니다. 간접은 주소 투명성을, 직접은 최적 경로를 제공합니다.

학습 팁: 주소 forwarding 우편 비유를 떠올리거나, 간접/직접 흐름도를 직접 그려보고, 키워드 카드 작성과 자문자답을 통해 개념을 정리해 보세요.
